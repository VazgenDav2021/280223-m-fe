1. Что такое фунцкия - функция это тот кусок кода который работает столько раз сколько раз мы вызвали данную фунцкцию, овновной причиной использования функции это минимизирование коприования кода, и с помощю функции код становится намного компактнее и читабльнее
2. В JS есть 3 вида обьявления функции:
   --1. FUNCTION DECLORATION - для создания функции данного типа надо написать ключевое слово "function", после этого написать имя функции (любое), после этого ставим (), в них можно указывать параметры которая данная функция принемает в качестве значения во время вызвоа, после этого ставим {}(тело функции) и в них пишем код который должен срабатывать каждый раз когда мы вызовим функцию
   --2. FUNCTION EXPRESSION - для создания функции данного типа надо написать ключевое слово "let,const,var" после это пишем название функции(любое), псоле этого ставим знак "=" после этого пишем ключевое слово "function", ставим (), в инх можно указывать параметры которая функция принемает в качестве значения во время вызвоа, после этого ставим {}(тело функции) и в них пишем код который должен срабатывать каждый раз когда мы вызовим функцию.
   --3. ARROW FUNCTION - для создания функции данного типа надо написать ключевое слово "let,const,var" после это пишем название функции(любое), псоле этого ставим знак "=" и ставим (), в инх можно указывать параметры которая функция принемает в качестве значения во время вызвоа, после ставим ставим "=>"(поэтому называется стролочная функция), и если тело функции состои из одной строки то можем не писать {} и слово "return", но если тело функции сосотои из более 1ой строки то ставим {}, и в них пишем тот код который должен срабатывать при каждом вызвое функции

3. Отличие
   --1. при использовании подхода FUNCTION DECLORTATION мы можем вызывать данную функцию до ее инициализации(создания). Если будем использовать FUNCTION EXPRESSION и ARROW FUNCTION то мы можем их вызывать только после создания. Если вызвать их до их инициализации то упдает ошибка (нельзя исползьвать ${НАЗВАНИЕ_ФУНКЦИИ} до ее инициализации).
   --2. Если у нас при вызове функции идут рандомные количество параметров(во врмея первого вызва 1 параметр, во ремя воторого 20, во время 3ого 5 параметров), то можем исползовать подоход создания функции через FUNCTION DECLORATION и FUNCTION EXPRESSION для того чтобы могли использовать такое значение как "arguments", значение "arguments" на возвращает динамичную коллкецию параметров при каждом вызове функции

4. RETURN
   --1. Очень часто мы создаем переменную которая ровна результату вызава функции, и в дальнейшем мы хотим храниь результат выполнения функции в переменной, поэтому надо использовать ключевое слово "return", это означает что функция нам ВОЗВРАЩАЕТ какое-то значение. console.log нам нечего не возвращает а просто выводит какое-то действие или значение в нашу консоль
   --2. После "return" не надо писать како-то код, поскольку после написания "reutrn" и значния, весь код игнорируется
   --3. Нам возращает одно значение, не может возращать несколько сразы
   --4. Можем в качестве результата возращать один массив или объект который содержит несколько значений

5. В качестве параметра может использоватся лобое значение (string,boolean,number, unfedined, function, array, object так далее)

6. Дефолтные параметры - дефолтные параметры это те значения которые присваываются параметрам во время инициализации функции, они служат для того, что если мы во время вызова не передали параметры, код работал по запланированному сценарию
